## 📚 WIT WT901BLECL — Référence Complète (Conservée Précieusement)

### Identification de l'appareil

```
Modèle : WitMotion WT901BLECL (WT901BLE68)
MAC BLE : E9:10:DB:8B:CE:C7
Nom BLE : WT901BLE68
Protocole : BLE 5.0 (principal) + UART série (backup)
Batterie : LiPo 1S (3.0V min → 4.2V max)
LED : bleue clignotante = connecté / éteinte = batterie morte
Port USB : /dev/ttyUSB1 (CH340 adapter, ID 1a86:7523)
Alias : /dev/ttyMidnightRider_IMU
```

### Structure du packet de données (20 bytes, header 0x55 0x61)

```
Byte 0-1 : 0x55 0x61 (header — toujours vérifier avant décodage)
Byte 2-3 : Accel X (int16 LE, /32768 × 16 × 9.81 → m/s²)
Byte 4-5 : Accel Y (int16 LE, /32768 × 16 × 9.81 → m/s²)
Byte 6-7 : Accel Z (int16 LE, /32768 × 16 × 9.81 → m/s²)
Byte 8-9 : Gyro X (int16 LE, /32768 × 2000 × π/180 → rad/s)
Byte 10-11 : Gyro Y (int16 LE, /32768 × 2000 × π/180 → rad/s)
Byte 12-13 : Gyro Z (int16 LE, /32768 × 2000 × π/180 → rad/s) = Rate of Turn
Byte 14-15 : Roll (int16 LE, /32768 × π → radians)
Byte 16-17 : Pitch (int16 LE, /32768 × π → radians)
Byte 18-19 : Yaw (int16 LE, /32768 × π → radians)
```

### Paths Signal K — format obligatoire

```
navigation.attitude.roll      (rad)  — Gîte (heel)
navigation.attitude.pitch     (rad)  — Assiette
navigation.attitude.yaw       (rad)  — Cap/Yaw
navigation.rateOfTurn        (rad/s) — Vitesse rotation (Gyro Z)
```

### Protocole de commandes WitMotion

```
Format : 0xFF 0xAA [registre] [data_L] [data_H]

Commandes importantes:
- 0xFF 0xAA 0x01 0x01 0x00 → Calibration accélérométrie (3 sec)
- 0xFF 0xAA 0x00 0x00 0x00 → Save config
- 0xFF 0xAA 0x65 0x00 0x00 → DFU mode (firmware update)
```
